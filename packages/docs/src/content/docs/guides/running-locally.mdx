## Prerequisites

- **Node.js 20+**
- **pnpm 11+** — install with `npm i -g pnpm`
- **PostgreSQL 15+** — running and accessible
- **[Expo Go](https://expo.dev/go)** on your phone, or an Android/iOS simulator

## 1. Install dependencies

From the monorepo root:

```sh
pnpm install
```

## 2. Set up environment variables

Copy the example file:

```sh
cp .env.example .env
```

Then edit `.env`. The **required** variables (the API won't boot without them) are:

| Variable | Description |
|---|---|
| `DATABASE_URL` | PostgreSQL connection string, e.g. `postgresql://user:pass@localhost:5432/musubi` |
| `BETTER_AUTH_SECRET` | Any long random string — used to sign sessions |
| `BETTER_AUTH_URL` | The **origin** of the API server, e.g. `http://localhost:7531`. **No trailing slash, no `/api` suffix.** |
| `ENVIRONMENT` | Set to `dev` during development — enables CORS for localhost |

`API_SERVER_PORT` is optional (defaults to `7531`). Everything else is **optional** — the API boots without it, and the feature that uses it simply won't work until set:

| Variable | Enables |
|---|---|
| `GOOGLE_WEB_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` | Google sign-in and Google Calendar sync |
| `MICROSOFT_CLIENT_ID`, `MICROSOFT_CLIENT_SECRET` | Outlook calendar sync (Microsoft Entra ID app registration) |
| `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, `FROM_EMAIL` | Password-reset (and other) emails |
| `CALDAV_ENC_KEY` | Connecting CalDAV accounts and storing federated Musubi connections (encrypts stored secrets) — generate with `openssl rand -hex 32` |
| `EXTERNAL_SYNC_INTERVAL_MIN` | Minutes between scheduled provider syncs (Google/Outlook/CalDAV) (live updates for members). Defaults to `5`; set `0` to disable |
| `LOG_LEVEL` | Server verbosity: `debug`, `info`, `warn`, `error`, or `silent`. Defaults to `info`; use `debug` when tracing sync |

<Aside type="caution">
`BETTER_AUTH_URL` must be the origin only — no `/api` suffix. Both the client and Better Auth append their own paths to it. If you set it to `http://localhost:7531/api`, requests will land at `/api/api/...` and fail.
</Aside>

## 3. Run database migrations

```sh
pnpm db:migrate
```

This applies all pending Drizzle migrations to your PostgreSQL database.

## 4. Start everything

From the monorepo root:

```sh
pnpm dev
```

Turborepo runs the API, the Expo client, and the docs site in parallel with an interactive TUI — each gets its own pane. Use `--ui=stream` for plain output.

To run a single piece in isolation:

```sh
cd apps/api && pnpm dev        # API only  → http://localhost:7531
cd apps/client && pnpm dev     # Expo client only
cd packages/docs && pnpm dev   # Docs site only → http://localhost:4321/docs
```

The Expo dev server will print a QR code. Scan it with Expo Go on your phone, or press `i` for an iOS simulator or `a` for Android.

<Aside type="tip">
If you're testing on a **real phone** (not a simulator), `localhost` won't work — your phone and computer are different machines. Set `BETTER_AUTH_URL` and the app's server URL to your machine's local network IP, e.g. `http://192.168.1.42:7531`. You can find it with `ip addr` (Linux) or `ifconfig` (macOS).
</Aside>
